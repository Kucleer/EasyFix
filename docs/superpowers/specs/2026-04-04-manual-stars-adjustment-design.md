# 积分手动调整功能设计

## 一、功能概述

在管理后台允许管理员手动调整用户积分（增加或减少），调整记录通过激励中心可查看。

## 二、功能位置

- **管理后台**：Management.vue - 激励Tab下新增"积分调整"区块
- **激励中心**：Motivation.vue - 积分明细中可查看所有调整记录及原因

## 三、API 设计

### 3.1 新增接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/stars/adjust | 手动调整积分 |

**请求体：**
```json
{
  "delta": 100,
  "reason": "管理员奖励"
}
```

**响应：**
```json
{
  "success": true,
  "new_balance": 150,
  "delta": 100,
  "record_id": 123
}
```

**校验规则：**
- `delta` 为 0 时返回错误
- 减少积分时，检查余额是否足够，不够返回错误
- `reason` 为必填，最长 200 字符

### 3.2 预设行为数据

在 `star_action` 表中预设一条记录：

| 字段 | 值 |
|------|-----|
| code | manual_adjustment |
| name | 手动调整 |
| star_value | 0 |
| is_preset | true |

## 四、数据模型

### star_record 表（现有）
| 字段 | 用途 |
|------|------|
| action_code | 手动调整时记录为 "manual_adjustment" |
| star_delta | 积分变动值（正或负） |
| reason | 调整原因 |

无需新增字段。

## 五、前端设计

### 5.1 管理后台 - 积分调整区块

**位置**：Management.vue - 激励Tab内，行为配置上方

**布局：**
```
┌─────────────────────────────────────┐
│ 积分调整                             │
├─────────────────────────────────────┤
│ 积分变动：[-] [+100] 输入框          │
│ 调整原因：[________________] (必填)  │
│                                     │
│        [确认调整]                    │
└─────────────────────────────────────┘
```

**交互：**
1. 输入积分变动值（支持负数）
2. 输入调整原因（必填）
3. 点击确认，调用 API
4. 成功后显示新余额，刷新积分明细

### 5.2 激励中心 - 积分明细

展示不变，`action_code = "manual_adjustment"` 时显示"手动调整"标签。

## 六、错误处理

| 错误 | 提示 |
|------|------|
| delta 为 0 | 积分变动不能为 0 |
| reason 为空 | 请输入调整原因 |
| 余额不足 | 积分不足，无法减少 |
| 余额为负 | 不允许调整为负数 |

## 七、实现步骤

1. 后端：新增 `POST /api/stars/adjust` 接口
2. 后端：在 `star_action` 表插入预设记录
3. 前端：Management.vue 新增积分调整表单
4. 前端：调用新接口并处理响应
