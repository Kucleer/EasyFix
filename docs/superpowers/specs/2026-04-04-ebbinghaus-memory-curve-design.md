# 艾宾浩斯记忆曲线复习功能设计

> **Goal:** 实现基于艾宾浩斯遗忘曲线的单词复习功能，根据记忆规律智能推荐复习时机，提升单词记忆效果。

---

## 一、核心数据结构

### 1.1 单词模型增强（Word）

新增字段：
- `learning_phase`（当前学习阶段）：新学/在途/遗忘点/牢记

已有字段复用：
- `ease_factor`：记忆牢固度（初始250）
- `interval`：当前间隔天数
- `next_review_at`：下次复习时间

### 1.2 复习记录表（WordReviewLog）

已有字段：word_id, is_correct, user_answer, reviewed_at

---

## 二、复习抽取算法

用户选择复习数量 N，按优先级抽取并补足：

```
1. 未复习单词（review_count=0）→ 取前 N 个
2. 若不足 N，取曲线到期单词（next_review_at ≤ today）→ 补满 N
3. 若仍不足 N，取低正确率单词（accuracy < 60%）→ 补满 N
4. 若仍不足 N，取随机其他单词 → 补满 N
```

---

## 三、记忆阶段定义

| 阶段 | 条件 | 进度条位置 |
|------|------|-----------|
| 新学 | review_count = 0 | 0-25% |
| 在途 | 1 ≤ review_count ≤ 2 | 25-50% |
| 遗忘点 | next_review_at ≤ tomorrow | 50-75% |
| 牢记 | 连续3次+正确且间隔≥7天 | 75-100% |

---

## 四、单词详情页 - 记忆曲线

### 4.1 阶段进度条

展示 4 阶段进度条，用圆点标记当前所处阶段：
- 新学 → 在途 → 遗忘点 → 牢记
- 当前阶段高亮显示

### 4.2 复习历史时间轴

按时间倒序展示每次复习记录：
- 正确：✓
- 错误：✗ 【用户回答】或 ✗ 【 - 】

```
复习历史：
● 今天（待复习）
● 4月6日 ✗ 【subtle】
● 4月3日 ✓
● 4月1日 ✓
```

### 4.3 下次复习时间

显示预测的下次复习时间：
- 若在遗忘点阶段：显示"即将到期"
- 其他阶段：显示具体日期

### 5.1 卡片替换

将"复习次数"卡片替换为"待复习"卡片：

| 原有 | 改为 |
|------|------|

```
待复习数量 = 未复习单词数 + 曲线到期单词数
```

## 六、API 变更

### 6.1 GET /api/words/{word_id}/memory-curve

获取单词的记忆曲线数据：
```json
{
  "word_id": 1,
  "learning_phase": "遗忘点",
  "interval": 4,
  "next_review_at": "2026-04-06T00:00:00",
  "review_history": [
    {
      "reviewed_at": "2026-04-06T10:00:00",
      "is_correct": false,
      "user_answer": "subtle"
    },
    {
      "reviewed_at": "2026-04-03T10:00:00",
      "is_correct": true,
      "user_answer": null
    }
  ]
}
```

### 6.2 POST /api/words/review/start

保持现有接口，优先按抽取算法返回单词列表。

### 6.3 GET /api/stats/home

新增返回 `word_stats.to_review_count`（待复习单词数）。

## 八、交互流程

```
1. 用户进入单词详情页
2. 点击"记忆曲线" tab
3. 查看当前阶段、进度条、复习历史
4. 用户可点击"去复习"按钮跳转到复习页
5. 复习完成后，历史记录更新
```

---

## 九、边界情况

1. 新单词（无复习历史）：只显示"新学"阶段，无历史记录
2. 单词错误无用户回答：显示 ✗ 【 - 】
3. 单词全部正确：阶段推进到"牢记"
4. 单词连续错误：interval 重置为 1，阶段退回"在途"
